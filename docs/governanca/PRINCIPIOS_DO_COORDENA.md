# PRINCIPIOS_DO_COORDENA

Data de instituicao: 2026-06-18

## Proposito

O Coordena existe para apoiar coordenacao humana, nao para produzir burocracia digital.

Seu papel e ajudar pessoas a:

- organizar demandas reais;
- manter memoria operacional;
- registrar encontros e decisoes;
- sustentar colaboracao cotidiana;
- reduzir atrito no trabalho de coordenacao.

## Principios permanentes

### 1. Coordenacao antes de controle

Toda funcionalidade deve fortalecer a articulacao entre pessoas, e nao apenas ampliar mecanismos de registro ou vigilancia.

### 2. Simplicidade antes de complexidade

Se uma solucao adiciona mais peso do que clareza, ela deve ser revista.

### 3. Mobile como ambiente principal

O sistema deve funcionar bem no celular, porque a coordenacao acontece em movimento, em contexto real e sob pressao cotidiana.

### 4. Memoria como infraestrutura

Decisoes, aprendizados e rastros importantes devem ser preservados de forma consultavel, sem exigir excesso de preenchimento.

### 5. Profundidade antes de expansao

Novos modulos so devem surgir quando os modulos atuais estiverem maduros o suficiente para sustentar crescimento com consistencia.

### 6. Linguagem acolhedora e operacional

A interface deve usar termos que organizem sem intimidar. O sistema deve soar como apoio, nao como cobranca.

## Protocolo para novas funcionalidades

Antes de implementar qualquer funcionalidade, responder:

1. Qual problema humano isso resolve?
2. Quem sera beneficiado?
3. Isso simplifica ou adiciona complexidade?
4. Isso funciona bem no celular?
5. Isso melhora a coordenacao entre pessoas?

Se a resposta para a quinta pergunta for negativa, a proposta deve ser reavaliada.

## Regra operacional permanente

Toda funcionalidade nova deve responder claramente a pelo menos um dos principios do Coordena.

Quando a funcionalidade, mudanca de nomenclatura ou decisao arquitetural alterar identidade, comportamento ou direcao do projeto, deve ser avaliada a necessidade de registro em:

- `docs/governanca/DECISOES_ESTRATEGICAS.md`

Essa regra existe para preservar coerencia, nao para ampliar burocracia.

Nao e necessario documentar tudo.

E necessario documentar aquilo que muda o sentido do projeto.
